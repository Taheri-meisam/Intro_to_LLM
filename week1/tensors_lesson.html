Part 01

What Is a Tensor?

A tensor is a multi-dimensional array. You already know some of these by other names ‚Äî scalars, vectors, matrices. Tensors generalize all of them.

PyTorch tensors are special because they can run on GPUs (thousands of operations in parallel) and track every operation performed on them so gradients can be computed automatically during backpropagation.

Part 05

Reshaping Tensors

The same 12 numbers can be arranged as a 1D vector, a 3√ó4 matrix, a 4√ó3 matrix, or even a 2√ó2√ó3 block. The data doesn't change ‚Äî only how it's organized.

‚ñ∏ Reshape Explorer ‚Äî 12 Elements
# Reshape examples
t = torch.arange(12)         # [0, 1, 2, ..., 11]
t.reshape(3, 4)             # 3 rows √ó 4 columns
t.reshape(4, 3)             # 4 rows √ó 3 columns
t.reshape(2, 2, 3)          # 3D: 2 blocks of 2√ó3

Squeeze removes dimensions of size 1. Unsqueeze adds them.

# squeeze: (1, 3) ‚Üí (3,)
t = torch.tensor([[1, 2, 3]])   # shape (1, 3)
t.squeeze()                      # shape (3,)

# unsqueeze: (3,) ‚Üí (1, 3)
t = torch.tensor([1, 2, 3])      # shape (3,)
t.unsqueeze(0)                   # shape (1, 3)

Part 06

Device Management ‚Äî CPU vs GPU

A GPU can perform thousands of matrix operations simultaneously, while a CPU handles them roughly one at a time. Neural network training involves millions of multiplications, so a GPU turns hours into minutes.

The catch: CPU and GPU have separate memory. A tensor on the CPU can't interact with one on the GPU ‚Äî you'd get an error. So you must be deliberate about where tensors live.

# Device-agnostic code (works with or without GPU)
device = torch.device('cuda' if torch.cuda.is_available() else 'cpu')

# Create on the right device
tensor = torch.randn(3, 3).to(device)

# Move between devices
gpu_tensor = cpu_tensor.to('cuda')
back_to_cpu = gpu_tensor.to('cpu')
‚ö†Ô∏è Common error: "Expected all tensors to be on the same device." This means you tried to combine a CPU tensor with a GPU tensor. Fix: move everything to the same device with .to(device).

Part 07

Data Types

PyTorch infers the type from your data. But sometimes you need to convert.

# PyTorch infers types
torch.tensor([1.0, 2.0])   # ‚Üí float32
torch.tensor([1, 2])       # ‚Üí int64
torch.tensor([True, False]) # ‚Üí bool

# Convert types
t = torch.tensor([1.5, 2.7, 3.9])
t.int()    # ‚Üí [1, 2, 3]  (truncates)
t.long()   # ‚Üí [1, 2, 3]  (int64)
t.float()  # ‚Üí stays float32
